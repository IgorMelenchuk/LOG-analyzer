# LOG-analyzer

### Описание проекта

**LOG-analyzer** - это легковесный консольный инструмент, написанный на Python, разработанный для парсинга, фильтрации и анализа логов. Он предназначен для мониторинга безопасности бэкенда, поддержки обнаружения потенциальных угроз безопасности (SQL-инъекций, XSS, Path Traversal и DDoS) и вывода отчетов о статистике по топ-5 IP-адресам, самым частым ошибкам и критическим инцидентам.

## Особенности проекта

### 1. Парсинг логов
Инструмент поддерживает парсинг логов из файла или через STDIN:

```bash
# Использование с файлом:
python main.py access.log

# Использование с(STDIN):
cat access.log | python main.py
```

### 2. Фильтрация по уровню критичности и HTTP-кодам ответа
Выборочно фильтруйте логи на основе уровня критичности (INFO, WARN, ERROR, CRITICAL) или по кодам HTTP отклика (4xx, 5xx):

```bash
python main.py access.log -l ERROR -s 404
```

### 3. SecOps-анализ безопасности
Встроенная система обнаружения аномалий на основе регулярных выражений включает следующие правила:
- SQL инъекции (SQLi)
- Перенаправление скриптов (Cross-Site Scripting, XSS)
- Просмотр несанкционированных путей (Path Traversal)
- Высокая активность с одного IP (подозрительно высокая активность, потенциальная DDoS или брутфорс)

### 4. Отчет в консоли
Отчет о логах извлекает топ-5 IP-адресов с наибольшим количеством запросов, топ-5 самых частых ошибок и общее количество критических инцидентов (4xx и 5xx). Вывод отчета в читаемом формате:

```
Top-5 IP Addresses:
[('127.0.0.1', 100), ('192.168.0.1', 60),...]

Top-5 Most Frequent Errors:
[('[Error]: Not Found (404)', 30), ('[Critical]: Internal Server Error (500)', 25)]

Total Critical Incidents: 110
```

## Зависимости

Для работы проекта требуется установка Python 3.9 и необходимые библиотеки, указанные в `requirements.txt`:

```bash
pip install -r requirements.txt
```

## Установка и запуск с помощью Docker

### Шаг 1: Сборка образа (Dockerfile)

С помощью файла Dockerfile создается контейнер, который устанавливает все зависимости и копирует код в рабочую директорию.

```bash
docker build -t loganalyzer .
```

### Шаг 2: Запуск контейнера

#### 2.1 Запуск с файлом логов:

Используйте Docker-volume для связи директории проекта внутрь контейнера и запустите скрипт на файле `access.log`.

```bash
docker run -v $(pwd):/app -it loganalyzer access.log
```

#### 2.2 Запуск с чтением STDIN:

Запуск через стандартный ввод STDIN:
  
```bash
cat access.log | docker run -i loganalyzer
```

#### 2.3 Общий запуск (без входных параметров):

Если не указан путь к файлу логов, инструмент будет читать данные из STDIN.

```bash
docker run -it loganalyzer
```

### Шаг 3: Тестирование внутри контейнера

Вы можете запустить тесты и проверить корректность работы проекта без необходимости устанавливать Python на сам сервер:

```bash
docker run it loganalyzer sh -c "python -m unittest discover"
```
